[教程]利用GitHub+picgo建立一个免费且稳定的图床


之前有位老哥写了篇gitee+picgo的教程，但是gitee会被审查，所以我写一篇PicGo + GitHub的教程。
PicGo + GitHub配置简单，使用优雅。背靠 GitHub 和微软，稳定性问题基本不用担心。还有就是支持 Windowns，macOS 和 Linux 平台。唯一的缺点，如果算的话，就是隐密性问题。因为所有图片都是上传到了 GitHub 的一个公有仓库，如果在意这点的话就不太适合。不过我上传的都是技术文章中的配图，这一点对我来说根本不是问题。第三方图床可能会存在图片失效，对翻旧帖的朋友不够友好，尤其是在盖区，更是难受。
下面就来教大家如何配置，非常简单。
配置 GitHub
新建仓库：


仓库得设置为 Public 。因为后面通过客户端访问算是外部访问，因此无法访问 Private ，这样的话图片传上来之后只能存储不能显示。
仓库建好之后，点击页面右上角头像，进入 Settings：


点进去后输入密码

填写 Notes 信息，选择 token 过期时间，为了安全，GitHub 会强烈建议不要设置成永久。这个大家根据自己实际情况选择，到期之后重新生成即可。
复选框的话，repo 一定要全选，其他的无所谓，我是都勾选了。
确定之后，就生成我们需要的 token 了。token 之显示一次，请认真保存。当然，可以重复上面的操作。
下载picgo
https://github.com/Molunerfinn/PicGo
